Contribute an Open Science Tip

Got an open science practice worth sharing? We’d love to feature it! Tips are community-contributed and curator-reviewed before appearing on the site.

Note💡 How to Submit
  1. Click the button below to open a pre-structured GitHub issue form
  2. Fill in the required fields — name, category, tip text, and a learn-more link
  3. Optionally include a thumbnail image URL (must be under 120 kB and publicly accessible)
  4. Submit — a curator will review and respond on the issue thread

Submit a Tip via GitHub →

What Happens After You Submit

Once submitted, a curator will review your tip and take one of three actions:

Decision What happens
✅ Accept Curator applies the tip-accepted label → tip is automatically added to the site
✏️ Modify & accept Curator edits the issue body, then applies tip-accepted
❌ Decline Issue is closed with a comment explaining why

You’ll be notified on the issue thread either way.

Tip Writing Guidelines

Tip📋 What makes a good tip?
  • Short name (3–5 words): memorable and easy to scan
  • Tip text (2–4 sentences): explain what it is, why it matters, and how to get started
  • Link: stable, authoritative source — official docs, OSF, institutional or journal sites preferred
  • Image (optional): a logo, icon, or illustration that represents the tip. Must be openly licensed (CC0, CC BY, or public domain) and under 120 kB

Image Guidelines

Warning🖼️ Image requirements
  • File size must be under 120 kB — larger images will be flagged automatically and excluded
  • Provide a direct URL to a publicly accessible image (e.g. hosted on GitHub, OSF, or Wikimedia Commons)
  • Accepted formats: PNG, JPG, WebP, GIF
  • Image must be openly licensed (CC0, CC BY, or public domain)

If your image is too large, tools like Squoosh can reduce file size without noticeable quality loss.

For Curators

Note🔧 Curator workflow
  1. Review the submitted issue for quality, accuracy, and fit with the project
  2. If edits are needed, modify the issue body directly before labeling
  3. Apply the label tip-accepted to trigger the automation
  4. The GitHub Action will:
    • Parse the issue fields
    • Validate and download any submitted image
    • Append the new tip to _code/tips.yaml
    • Commit the changes and trigger a site re-render and deploy
    • Post a confirmation comment and close the issue

If the automation cannot parse a field, it will comment on the issue with the specific error rather than failing silently. Re-apply the tip-accepted label after fixing to retry.
